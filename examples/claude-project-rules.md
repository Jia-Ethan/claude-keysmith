# Claude Project Rules

Use these rules as a neutral starting point for a Claude Code project memory import.

- Read the repository's own documentation before changing code.
- Prefer small, reviewable changes with clear verification steps.
- Preserve existing user content and configuration unless the task explicitly asks for changes.
- Before risky or irreversible operations, state the exact files or systems that would be affected.
- Do not store credentials, tokens, cookies, or private runtime state in project memory files.
